# Анализ результатов проекта

## 1. Общие показатели
В базе содержится 59 клиентов и 412 заказов.  
Общая выручка сервиса составляет 2328.60, а средний чек — 5.65.  
Минимальная сумма заказа заметно ниже средней, а максимальная — существенно выше, что говорит о разбросе сумм отдельных покупок.

Отдельно был рассчитан показатель ARPU:
- ARPU = 39.47

Это означает, что в среднем один уникальный клиент принёс сервису 39.47 единиц выручки за рассматриваемый период.

## 2. Динамика заказов и выручки
Была рассчитана выручка по дням, количество заказов по дням и средний чек по дням.

Анализ показал, что в большинстве дат на один день приходится один заказ, хотя в отдельных случаях встречаются даты с двумя заказами.  
Это объясняет, почему в ряде случаев динамика выручки по дням визуально близка к динамике суммы отдельных заказов.

Таким образом, датасет не показывает интенсивный поток транзакций внутри одного дня, а скорее отражает последовательную историю покупок.

## 3. Географический анализ
Отдельный блок анализа был посвящён странам.

По выручке лидируют:
1. USA — 523.06
2. Canada — 303.96
3. France — 195.10
4. Brazil — 190.10
5. Germany — 156.48

По числу заказов также лидирует USA, далее идут Canada, Brazil, France и Germany.

Это говорит о том, что именно рынок США даёт основной вклад как по объёму заказов, так и по итоговой выручке.

По среднему чеку среди стран лидируют Chile, Hungary, Ireland и Czech Republic.  
Это значит, что некоторые страны не являются лидерами по объёму продаж, но отличаются более высокой средней стоимостью одного заказа.

## 4. Анализ клиентов
Был выполнен расчёт топ-клиентов по выручке и по числу заказов.  
Этот блок позволяет выделить наиболее ценных клиентов сервиса.

Также был рассчитан ARPU, который показывает среднюю выручку на одного клиента и позволяет оценить общую монетизацию пользовательской базы.

## 5. Сегментация клиентов по количеству заказов
Для сегментации были заданы три группы:
- 1 order group
- 2-5 order group
- 6+ order group

После агрегации выяснилось, что практически все клиенты попадают в группу `6+ order group`.  
Это означает, что в данном датасете клиентская база очень однородна по количеству заказов: большинство пользователей делают много покупок, а клиентов с единичными заказами почти нет.

Такой результат важен для интерпретации:
- сама логика сегментации реализована корректно, однако конкретный датасет не даёт ярко выраженного разделения клиентов по активности

## 6. Выводы
По итогам анализа можно сделать несколько выводов:

1. Основной рынок сервиса — USA: страна лидирует и по количеству заказов, и по выручке.
2. Средний чек по базе относительно стабилен и составляет 5.65.
3. Один клиент в среднем приносит 39.47 единиц выручки.
4. Клиентская база в данном датасете отличается высокой однородностью по количеству заказов.
5. Датасет хорошо подходит для демонстрации навыков SQL-анализа, даже несмотря на ограниченную вариативность сегментации.

При дальнейшем развитии будет добавлено:
- анализ продаж по артистам и жанрам
- анализ состава заказов через `invoice_line`
- выделение топовых артистов по выручке
- анализ распределения треков по альбомам и жанрам
- более сложную сегментацию клиентов с использованием оконных функций
